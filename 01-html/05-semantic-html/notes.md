# Semantic HTML Notes

## Important Concepts & Tags
- `<header>`: Represents introductory content or navigational aids.
- `<nav>`: Defines a block of navigation links.
- `<section>`: Defines a thematic group of content.
- `<article>`: Defines self-contained, independent article content.
- `<footer>`: Defines footer content (e.g., contact info, copyright).

## Common Mistakes
- Using generic `<div>` elements for everything, losing structural meaning.
- Wrapping document content in multiple nested `<section>` elements without headings (`<h1>`-`<h6>`).

## Interview Notes
- **Why should developers use Semantic HTML over standard divs?** Semantic HTML improves Search Engine Optimization (SEO), enhances accessibility for screen readers, and makes the code cleaner and easier to maintain for developers.

## Practice Ideas
- Refactor an old-style portfolio page consisting only of nested divs into a semantic header-main-nav-footer structure.
